# Решение: GM810 UART не получает события

## ✅ Проблема найдена и исправлена

**Дата:** 2026-04-24  
**Статус:** ИСПРАВЛЕНО  
**Тип:** КРИТИЧЕСКАЯ

---

## 🔴 Что было не так

**GPIO пины PA11/PA12 для USART6 не инициализировались!**

Хотя USART6 контроллер был инициализирован в `MX_USART6_UART_Init()`, физические GPIO пины PA11 и PA12 никогда не были подключены к функции USART6_TX / USART6_RX.

**Результат:**
- STM32 контроллер UART6 готов, но пины не подключены
- Данные с GM810 модуля никогда не попадают на пины PA11/PA12
- Нет UART прерываний
- `gm810_restart_receive_it()` никогда не вызывается
- `service_gm810_uart_rx_callback()` не получает события

---

## ✅ Что было сделано

### Исправление: добавлена GPIO инициализация в `Core/Src/main.c`

**Файл:** `Core/Src/main.c`  
**Функция:** `MX_GPIO_Init()` (строки 836-850)

```cpp
/* USER CODE BEGIN GM810_USART6_GPIO_INIT */
#if HW_PROFILE_GM810_USART6
  /* Configure GPIO pins for USART6 (GM810 QR reader on PA11/PA12)
   * PA11: USART6_TX (Alternate Function 8)
   * PA12: USART6_RX (Alternate Function 8) */
  GPIO_InitStruct.Pin = GPIO_PIN_11 | GPIO_PIN_12;
  GPIO_InitStruct.Mode = GPIO_MODE_AF_PP;
  GPIO_InitStruct.Pull = GPIO_NOPULL;
  GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_VERY_HIGH;
  GPIO_InitStruct.Alternate = GPIO_AF8_USART6;
  HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);
#endif
/* USER CODE END GM810_USART6_GPIO_INIT */
```

**Что это делает:**
- Вкл./выкл. по флагу `HW_PROFILE_GM810_USART6`
- Конфигурирует PA11 как USART6_TX (AF8)
- Конфигурирует PA12 как USART6_RX (AF8)
- Использует максимальную скорость для надежности

## ⚠️ Важные замечания

### 1. Условная компиляция
Исправление включено в условной компиляции:
```cpp
#if HW_PROFILE_GM810_USART6
```

Если флаг = 0, инициализация USART6 GPIO не выполняется (для совместимости с USB профилем).

### 2. USB конфликт
При `HW_PROFILE_GM810_USART6 = 1`:
- USB_OTG_FS **отключен** (HW_PROFILE_USB_OTG_FS = 0)
- PA11/PA12 используются для USART6
- USB функционала нет (как и предусмотрено планом)

### 3. STM32CubeMX регенерация
⚠️ **ВНИМАНИЕ:** Если код регенерируется из .ioc файла, эта GPIO инициализация может быть перезаписана!

**Рекомендация:** 
- Либо защитить код в `/* USER CODE BEGIN/END GM810_USART6_GPIO_INIT */`
- Либо добавить USART6 правильно в .ioc файл через CubeMX GUI

### 4. Полнота решения
Текущее решение:
- ✅ Быстрое
- ✅ Не требует переконфигурации CubeMX
- ✅ Явно понятно
- ⚠️ Может быть потеряно при регенерации

Рекомендуется позже (когда время позволит):
