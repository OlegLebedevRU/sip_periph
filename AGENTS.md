## Project Overview

**sip_periph** вЂ” firmware for the **STM32F411CEU** microcontroller acting as a peripheral board ("SIP periph"). Responsibilities:

- Communicates with a host controller over **IВІC1** (slave role, outbox/inbox register protocol).
- Drives peripheral hardware over **IВІC2** (master role): TCA6408A I/O expander, DS3231M RTC, PN532 NFC reader.
- Provides a **USB CDC** (virtual COM port) for debug/HMI console output.
- Runs a **UART** interface to the DWIN display (`app_uart_dwin*`).
- Manages a small **SSD1306 OLED** display and a matrix keyboard.
- Built on **FreeRTOS** (CMSIS-RTOS v1 wrappers) on top of STM32 HAL.

Key IC neighbours: **GM810** (touch controller via UART), **PN532/UID532** (NFC), **TCA6408A** (8-bit I/O expander), **DWIN** (serial display), **DS3231M** (RTC), **SSD1306** (OLED).

See also: [`ARCHITECTURE.md`](ARCHITECTURE.md), [`GLOSSARY.md`](GLOSSARY.md), [`docs/README.md`](docs/README.md).

## Build & Toolchain

### Primary: STM32CubeIDE

1. Open STM32CubeIDE.
2. *File в†’ Import в†’ Existing Projects into Workspace* вЂ” select this directory.
3. Build with **Ctrl+B** (or *Project в†’ Build All*).
4. Flash via ST-Link using the built-in debug/flash functionality.

Project descriptor files (`.project`, `.cproject`) are tracked in git.

### Alternative: `make`

```bash
# Requires arm-none-eabi-gcc on PATH
make all        # build
make clean      # clean artefacts
```

Output artefacts (`*.elf`, `*.bin`, `*.hex`, `*.map`) are produced under `Debug/` or `Release/` and are excluded from git.

### OpenOCD (command-line flashing)

```bash
openocd -f "sip_periph.cfg"
# or for Release build:
openocd -f "sip_periph Release.cfg"
```

## What NOT to Touch (Without Explicit Request)

| Path / file | Reason |
|---|---|
| `Drivers/` | STM32 HAL + CMSIS вЂ” regenerated by CubeMX |
| `Middlewares/ST/` | ST middleware stack вЂ” regenerated by CubeMX |
| Most of `USB_DEVICE/` | Generated by CubeMX |
| `STM32F411CEUX_FLASH.ld`, `STM32F411CEUX_RAM.ld` | Linker scripts вЂ” tied to silicon |
| `sip_periph.ioc`, `.mxproject` | CubeMX project descriptors |

**In CubeMX-generated `.c`/`.h` files**, only write code inside the designated blocks:

```c
/* USER CODE BEGIN <section_name> */
// Your code here
/* USER CODE END <section_name> */
```

Everything outside these blocks will be **overwritten** by the next CubeMX re-generation.

## Where to Write Code

- **Own application modules** in `Core/Src/` and `Core/Inc/` вЂ” fully writable, no CubeMX constraints. Convention: name files `service_<device>.c` / `service_<device>.h` for peripheral drivers.
- **Inside USER CODE blocks** in CubeMX-generated files (e.g., `main.c`, `stm32f4xx_it.c`).

## Coding Conventions

| Rule | Detail |
|---|---|
| Language standard | C99 / C11 |
| Dynamic memory | **Forbidden** вЂ” no `malloc`/`free`. Use static buffers and FreeRTOS objects only. |
| ISR-safe API | In interrupt handlers use only FreeRTOS functions with `FromISR` suffix (e.g., `xQueueSendFromISR`). |
| Module prefix | Application service drivers: `service_<name>_*` (e.g., `service_tca6408_init`). |
| IВІC guard | IВІC2 shared bus access must be protected with `i2c2_MutexHandle`. |
| No bare delays | Do not use `HAL_Delay` inside tasks вЂ” use `osDelay` / `vTaskDelay`. |

## Workflow for PRs

1. Target branch: the repository's **default branch** (`main`).
2. Before opening a PR verify the project builds (CubeIDE or `make all`).
3. If you change a device communication contract, update the corresponding `docs/i2c_*_contract.md`.
4. **Do not commit** `Debug/`, `Release/`, `*.elf`, `*.bin`, `*.hex`, `*.map`.
5. Keep CubeMX-generated files intact outside USER CODE blocks.
